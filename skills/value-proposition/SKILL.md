# Ad-Lib 价值主张速写

使用 `display_diagram` 的 **content-fill 模式** 生成价值主张画布，把价值主张压缩成一句可测试的定位句。

## 何时使用

- 用户要求"价值主张"、"价值主张画布"、"Ad-Lib 价值主张"、"定位句"
- 用户想用一句话说清楚产品为谁、解决什么、有何不同
- 在完成价值主张画布（VPC）或 JTBD 分析后，需要收敛成可测试的表达

## 工具调用（content-fill 模式）

**必须使用 content-fill 模式，不要传 raw XML。**

调用 `display_diagram` 时传以下参数：
- `template_id`: `"value_proposition"`
- `title`: 画布标题
- `content`: JSON 对象，6 个 key 对应 6 个模块

### content keys:
| key | 模块 |
|-----|------|
| `product_service` | 01 产品与服务 |
| `customer_segments` | 02 客户细分 |
| `customer_jobs` | 03 客户任务 |
| `pain_relievers` | 04 痛点缓解 |
| `gain_creators` | 05 收益创造 |
| `competitors` | 06 竞争价值主张 |

### 调用示例:
```json
{
  "template_id": "value_proposition",
  "title": "某产品 价值主张速写",
  "content": {
    "product_service": "• 在线财务报表分析平台\n• AI 驱动的数据可视化\n• 自然语言查询接口",
    "customer_segments": "• 中小企业财务经理\n• 需要快速生成报表的非财务背景管理者",
    "customer_jobs": "• 快速生成季度财务报表\n• 向管理层汇报经营状况\n• 发现财务异常并定位原因",
    "pain_relievers": "• 消除手工 Excel 制表的繁琐\n• 减少报表格式错误导致的返工\n• 避免 SQL 依赖让非技术人员自助查询",
    "gain_creators": "• 加速报表生成从天级到分钟级\n• 实现自然语言对话式分析\n• 增加数据可视化覆盖面",
    "competitors": "• Excel + 手工制表\n• 传统 BI 工具（Tableau/Power BI）\n• 外包给财务咨询公司"
  }
}
```

### 内容格式规则:
- 用 `\n` 换行
- 用 `• ` 作为列表项前缀
- 每个模块内容不超过 300 字符
- 每个模块内容必须唯一

## Ad-Lib 句式

模板顶部固定展示 Ad-Lib 句式：
```
我们的 [产品/服务] 帮助 [目标客户]，他们想要完成 [客户任务]。
我们通过 [痛点缓解]，并 [收益创造] 来创造价值，
不同于 [竞争替代方案]。
```

## 六个组成部分

1. **产品与服务** — 客户真正会购买、使用或接触到的提供物
2. **客户细分** — 只服务一个具体人群，不服务「所有人」
3. **客户任务** — 客户雇用某个方案要完成的功能性、社会性或情感性任务
4. **痛点缓解** — 用一个动词说明如何减少、消除或避免客户最在意的痛点
5. **收益创造** — 用一个动词说明如何增加、实现或加速客户最想要的收益
6. **竞争价值主张** — 客户今天已经在用的替代方案（竞品、手工变通、内部流程或什么都不做）

## 分析要点

生成画布时应覆盖六个模块：

1. **产品与服务** — 列出客户真正会接触到的具体提供物（功能、服务、平台）
2. **客户细分** — 写出具体人群标签，避免「企业用户」等过宽描述
3. **客户任务** — 写客户想达成的结果，不是「需要工具」
4. **痛点缓解** — 用动词开头：减少/消除/避免 + 具体痛点
5. **收益创造** — 用动词开头：增加/实现/加速 + 具体收益
6. **竞争价值主张** — 写出客户当前的替代方案，说明差异化

## 好句子的标准

- **清楚**：5 秒内能读懂，不依赖行话或夸张词
- **具体**：客户、任务、痛点、收益都能被观察或验证
- **聚焦**：只服务一个细分，不试图覆盖所有人
- **有对比**：明确不同于客户当前的替代方案
- **可测试**：能转化成访谈问题、实验假设或购买意愿测试

## 使用建议

- 先从价值主张画布或客户访谈中提取素材，不要凭空写营销话术
- 一次只写一个客户细分和一个主要任务
- 至少写 3–5 个版本：换客户、换任务、换痛点、换收益或换竞争替代方案
- 比较每个版本：是否具体？是否重要？是否有差异化？是否能被验证？
- 把最强的一两个版本拿去做访谈、着陆页、销售话术或小实验

## 常见误区

- 把「客户细分」写成「企业用户」这类过宽标签
- 把「客户任务」写成「需要一个工具」，而不是客户想达成的结果
- 只写收益，不写痛点，导致价值主张缺少迫切性
- 只写痛点，不写收益，导致价值主张显得消极或不完整
- 忘记竞争替代方案，无法说明为什么客户要从现有做法切换
