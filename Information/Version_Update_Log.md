# 版本开发日志

## v1.2.0 LTS
**2026年7月17日**

本次长期支持版本聚焦 **更顺畅的协作分析、可编辑的商业图表，以及更可靠的数据处理体验**。

### 1. AI 团队协作升级
- 将一个复杂目标拆分为可追踪的协作任务，清楚展示进度、结果与下一步
- 可先确认执行计划，再按依赖关系推进；遇到需要人工决定的内容会明确提示
- 支持质量复核与针对性重试，让需要调整的环节可以单独处理，不必从头开始

### 2. 全新商业画布
- 新增商业模式画布、BCG 矩阵、SWOT 分析和空白画布，帮助把分析结论快速整理成图
- 可直接在应用内编辑图形、保存项目、重命名、回看和导出图片或 `.drawio` 文件

### 3. 更贴合业务的分析助手
- 技能会根据你的问题自动匹配，也可以创建和编辑自己的分析流程
- 支持生成可离线打开的交互式业务看板，便于分享分析结果
- 数据读取与分析提示更完善，减少字段、口径和图表不匹配带来的困扰

### 4. 更可靠、更高效的处理体验
- 大型任务与多项操作的处理更流畅，等待期间仍可清楚了解进度
- 工作目录中的常见文档和文件夹内容更容易读取与使用
- 数据分析结果增加了更清晰的质量提示和可核查依据，帮助判断结论是否可靠

### 5. 界面与使用体验改进
- 存储设置和窄屏界面布局更清晰，信息与操作不再拥挤
- 检查更新后可直接前往 GitHub Releases 选择所需安装包

### 6. 本地数据清理更安心
- 在“设置 → 存储”中查看本地数据占用、可清理候选和回收站内容
- 不再需要的上传文件和历史产物可先移入回收站；恢复或永久删除都需明确确认，减少误删风险
- 工作目录、知识库和仍被对话引用的数据会受到保护，不会被当作清理对象

## v1.1.0 LTS

**2026年7月2日**

本次升级新增 **斜杠命令系统、文件时光回退、Hooks 自动化、AI 预填建议** 等多项功能，同时全面提升聊天界面与 Dashboard 的交互体验。


### 1. 全新斜杠命令系统
- 全面重构斜杠命令，支持 `/new`（新会话）、`/stop`（停止响应）、`/data`（数据预览）、`/jobs`（任务历史）、`/compact`（压缩上下文）等系统命令
- 命令支持别名搜索、Tab 补全和参数提示；输入 `/help <命令名>` 可查看详细说明
- 新增自定义 Prompt 命令：在工作目录 `.baa/commands/` 下放置 Markdown 文件即可添加自定义命令

### 2. Hooks 用户自动化
- 新增 Hooks v1，支持在 AI 会话的关键节点（开始对话、每轮请求前后、工具调用前后）触发自动化动作
- 在应用设置中可直接编辑、校验和测试 Hooks 配置；支持 `prompt` 注入和 Python 脚本调用
- Agent 可读取公开文档或 API 说明，自动帮你生成 Hooks 配置

### 3. AI 预填下一步建议
- 每次 AI 回复完成后，自动预测并预填一条"下一步可发送的消息"到输入框
- 输入框为空时才预填，一旦开始输入、发消息或切换会话，旧建议自动失效

### 4. 工作目录时光回退
- 每次发消息前自动创建快照；工作目录中的文件被修改前也会自动备份旧版本
- `/checkpoint` 打开时间线面板，显示每个快照的触发消息和修改文件数
- 支持三种回退模式：恢复文件+对话、仅对话、仅文件，满足不同撤销需求

### 5. 工作目录管理增强
- 弹层新增历史记录列表，可一键切换到之前用过的工作目录
- 新增"移除记录"功能，从列表隐藏目录记录，本地文件全部保留
- 新增存储清理功能，自动识别并清理失效缓存，释放占用空间

## v1.0.0_LTS
**2026年6月20日**

首个长期支持版本（LTS），整合 v5.1 之后的全部改进，聚焦 **工作目录、分析技能、前端体验与安全加固**。

### 1. 工作目录挂载（全新）
- 可直接挂载本地文件夹作为工作目录，无需逐个上传文件即可分析其中的数据
- 挂载后自动注册目录内的 CSV / Excel 文件，效果等同批量上传，AI 可直接查询
- **秒级恢复**：关闭软件后已注册的表会保留，下次重新挂载无需重新解析，大 Excel 只解析一次
- 导出文件自动写入工作目录下的 `artifacts/` 文件夹，跟随项目统一管理
- 文件目录采用摘要、分页与按需搜索，避免大量文件撑满对话上下文

### 2. 分析技能系统
- 新增斜杠命令技能系统，可通过 `/funnel-analysis` 等命令快速调用预设分析流程
- 支持加载自定义技能，技能只注入提示词，不执行外部代码

### 3. 前端体验全面升级
- 界面核心渲染路径迁移至 Vue 框架，交互更流畅、响应更稳定
- 模型设置、知识库、MCP 工具管理三大面板重构，告别重复点击与状态错乱
- 工具执行间隙显示"正在思考下一步"等待态，不再误以为卡住
- 修复思考过程与工具步骤之间的动画空白

### 4. 历史对话管理
- 已保存对话支持重命名
- 加载历史对话时显示进度遮罩与耗时计数，可中途终止

### 5. 安全加固
- SQL 查询新增路径白名单与语法校验，防止越权读取工作目录之外的文件
- 网络地址（http / s3 等）一律拒绝，杜绝 SSRF 风险
- 文件读写强制经过工作目录鉴权

### 6. 项目结构整理
- 运行模块集中至 `infrastructure/`，用户文档与教程集中至 `Information/`

## v5.0
**2026年6月4日**

本次为一次大版本迭代，涵盖 **多数据源、智能交互、稳定性修复与安全加固** 四大方向。

### 1. 多数据源支持
- 现在可以同时连接多个数据源，不再限制只能用一个
- 侧边栏新增数据源列表，每个来源都有独立的开关，可以自由选择激活哪些数据源参与分析
- AI 在多数据源场景下能自动识别每条查询应该去哪个数据源取数，支持跨数据源的联合分析

### 2. AI 主动提问
- 当用户的需求比较模糊时（比如没有说清楚要按哪个维度分析、用哪个指标），AI 会主动暂停并提出问题
- 界面上会出现选项卡片，点击选项即可继续；也可以选"其他"输入自定义答案
- AI 不会随意打断——只有在不同选择会导致实质不同结果时才会提问

### 3. 对话自动保存
- 每次 AI 回复完成后会自动保存对话记录，刷新页面或重新打开也不会丢失
- 已保存对话列表中，自动保存和手动保存的记录样式统一，用小标签区分，不再有视觉上的"高低之分"
- 修复了加载已保存对话后再继续聊天时，自动保存会错误创建新记录的问题

### 4. MCP 工具接入体验优化
- 已接入的 MCP 工具现在可以展开查看详细说明，了解每个工具的用途和参数
- 新增"智能填充"功能：把工具的安装命令或配置粘贴进来，自动解析填好表单，不需要手动填写各个字段
- 新增"本地扫描"功能：填入本地工具包的路径，自动识别配置信息

### 5. 数据预览升级
- 数据预览弹窗改为左右分栏布局：左侧显示所有表的列表（含行数），右侧查看具体内容
- 多数据源时左侧会按来源分组，每组显示该来源包含多少张表
- 分隔栏可以拖动调整宽度

### 6. SQL 数据库连接改善

- 连接外部数据库后，AI 查询过的表会自动缓存到本地，后续重复查询速度大幅提升，无需每次都连远端
- 修复了表数量识别不准的问题（之前大型数据库可能只识别到部分表，现在能获取全部表和视图）
- 修复了跨数据库方言（MySQL / PostgreSQL 等）的表名引号格式错误
- 大表（超过 50 万行）不再全量拉取到本地，查询直接发往远端执行，避免内存溢出

### 7. 防止 AI 编造数据

- 修复了多轮对话中 AI 会"记住"之前回复里的数字、下一轮直接复用而不重新查询的问题
- 现在 AI 能看到完整的历史查询记录，不会把文字描述当成已知事实来使用

### 8. 知识库触发修复

- 修复了业务知识库在对话中经常不被触发的问题——之前 AI 会因为各种理由跳过查知识库这一步
- 现在只要是数据分析请求，AI 会优先查阅知识库中的指标定义和业务规则，再开始分析

### 9. 其他体验修复

- AI 回复时，用户向上翻看之前的内容不会再被强制拉回底部；新消息发出时才会自动滚到最新
- 修复了 Google Sheets 数据源有时只识别到一半工作表的问题
- 修复了分组柱状图的分组字段有时不生效、退化为普通柱状图的问题
- 修复了加载已保存对话后，侧边栏数据源状态不刷新、显示旧状态或空白的问题
- 增强了安全性：AI 执行的所有 SQL 查询经过更严格的语法层检查，并在数据库连接层面关闭了文件系统和网络访问权限

## v5.1
**2026年6月10日**

本次升级聚焦于 **多数据源联合分析、数据接入稳定性体验**。

### 1. 跨数据源联合分析增强
- 新增多数据源虚拟合并层，支持直接对不同数据源中的表执行跨源 `JOIN` 和 `UNION`
- 当多个 Excel 或 CSV 文件包含同名表时，系统会自动为表添加数据源前缀，AI 可以准确区分并查询指定文件
- 优化多数据源查询路由，单源查询与跨源查询均可自动选择正确的执行方式
- 数据源结构信息现在会附带样本数据，帮助 AI 区分结构相同但内容不同的表

### 2. Excel 数据接入修复
- 修复包含银行账号、证件号、编码等“空格分隔数字串”的 Excel 列上传失败问题
- 普通文本列会被稳定识别为字符串，不再被 DuckDB 错误推断为数值并触发类型转换异常

### 3. 数据源断连提示优化
- 修复服务重启后前端仍显示数据源已连接、查询却无响应的问题
- 当会话中的数据源连接已经失效时，系统会明确提示用户重新连接，不再出现空白回复
- 增加前后端双重检查，避免 AI 在缺少数据结构时继续无效执行

### 4. 模型测试体验优化
- 模型设置支持“先测试、后保存”
- 用户填写新的 API Key、Base URL 或模型名称后，可直接测试当前输入内容
- 输入框为空时仍会自动使用已保存配置，兼容原有测试方式
